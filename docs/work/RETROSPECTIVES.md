# Retrospectivas de Sprint

Notas al final de cada sprint: que salio bien, que mejorar, y compromisos de mejora.

---

## Template

```markdown
## Sprint [N] — [fecha inicio] → [fecha fin]

### Lo que salio bien
- ...

### Lo que puede mejorar
- ...

### Compromisos para el siguiente sprint
- ...

### Metricas
- Tareas completadas: X / Y
- Bugs encontrados: N
- Deuda tecnica generada: [descripcion breve]
```

---

## Sprint 2 — Marzo 2026

### Lo que salio bien
- Se implemento el flujo completo de autenticacion con Supabase Auth
- Ideas Board funcional: proponer, votar y eliminar ideas propias
- Hero Section rediseñada con estetica premium (glassmorphism, animaciones)
- Panel admin funcional para gestionar eventos, noticias, recursos y oportunidades
- Inscripcion a eventos con notificacion por email (Resend)
- Sistema de perfil con avatar generativo y banner personalizable

### Lo que puede mejorar
- La planificacion de tareas en `backlog.md` fue demasiado informal (todo en mayusculas, sin estimaciones)
- No se usaron branches por feature consistentemente
- La documentacion tecnica no se actualizo durante el sprint

### Compromisos para Sprint 3
- Mantener `STATUS.md` actualizado en cada sesion
- Usar prefijos de commit convencionales (`feat:`, `fix:`, etc.)
- Actualizar `ARCHITECTURE.md` cuando cambie el schema de Supabase
